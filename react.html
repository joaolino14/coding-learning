Pengenalan React

React adalah library JavaScript untuk membangun antarmuka pengguna (UI) yang efisien dan fleksibel.

Kelebihan React:

  • Komponen: UI dibangun dari komponen yang dapat digunakan kembali
  • Virtual DOM: Meningkatkan performa aplikasi
  • Uni-directional Data Flow: Aliran data yang mudah dipahami
  • Ekosistem Kaya: Banyak library pendukung

Kapan Menggunakan React?

  • Aplikasi web yang kompleks dengan banyak interaksi
  • Aplikasi yang membutuhkan pembaruan UI secara real-time
  • Proyek yang membutuhkan komponen reusable

Contoh Kode React Sederhana:

function Hello() {
  return <h1>Hello, React!</h1>;
}

ReactDOM.render(
  <Hello />,
  document.getElementById('root')
);

JSX

JSX adalah sintaks extension untuk JavaScript yang memungkinkan penulisan HTML dalam kode JavaScript.

Karakteristik JSX:

  • Bukan string maupun HTML
  • Produk dari React.createElement()
  • Memiliki aturan khusus:
    • Nama atribut camelCase (className bukan class)
    • Harus memiliki satu root element
    • Tag harus ditutup dengan benar (<img />)

Contoh JSX:

const element = (
  <div className="greeting">
    <h1>Hello, world!</h1>
    <p>Welcome to React</p>
  </div>
);

Embedding Expressions:

const name = 'John';
const element = <h1>Hello, {name}</h1>;

// Ekspresi JavaScript dalam JSX
const user = { firstName: 'John', lastName: 'Doe' };
const element = (
  <h1>
    Hello, {user.firstName} {user.lastName}
  </h1>
);

Komponen React

Komponen adalah blok bangunan dasar dari aplikasi React.

Jenis Komponen:

  • Functional Components (modern) 
    function Welcome() {
  return <h1>Hello, React!</h1>;
}
  • Class Components (legacy) 
    class Welcome extends React.Component {
  render() {
    return <h1>Hello, React!</h1>;
  }
}

Komposisi Komponen:

function App() {
  return (
    <div>
      <Welcome />
      <Welcome />
      <Welcome />
    </div>
  );
}

Export/Import Komponen:

// Welcome.js
function Welcome() {
  return <h1>Hello!</h1>;
}
export default Welcome;

// App.js
import Welcome from './Welcome';

Props

Props (properties) adalah cara untuk mengirim data dari komponen parent ke child.

Menggunakan Props:

function Welcome(props) {
  return <h1>Hello, {props.name}</h1>;
}

// Penggunaan
<Welcome name="Sara" />
<Welcome name="John" />

Children Props:

function Card(props) {
  return (
    <div className="card">
      {props.children}
    </div>
  );
}

// Penggunaan
<Card>
  <h2>Judul</h2>
  <p>Isi konten</p>
</Card>

Type Checking dengan PropTypes:

import PropTypes from 'prop-types';

function Welcome(props) {
  return <h1>Hello, {props.name}</h1>;
}

Welcome.propTypes = {
  name: PropTypes.string.isRequired
};

State

State adalah data yang dapat berubah selama komponen hidup dan memicu re-render ketika diubah.

State dalam Class Component:

class Counter extends React.Component {
  constructor(props) {
    super(props);
    this.state = { count: 0 };
  }

  render() {
    return (
      <div>
        <p>Count: {this.state.count}</p>
        <button onClick={() => 
          this.setState({ count: this.state.count + 1 })
        }>
          Increment
        </button>
      </div>
    );
  }
}

State dalam Functional Component (useState Hook):

import { useState } from 'react';

function Counter() {
  const [count, setCount] = useState(0);

  return (
    <div>
      <p>Count: {count}</p>
      <button onClick={() => setCount(count + 1)}>
        Increment
      </button>
    </div>
  );
}

Aturan State:

  • Jangan modifikasi state secara langsung
  • State updates mungkin asynchronous
  • State updates digabungkan

Lifecycle

Lifecycle methods memungkinkan eksekusi kode pada tahap tertentu dalam siklus hidup komponen.

Lifecycle Class Component:

  • Mounting:
    • constructor()
    • render()
    • componentDidMount()
  • Updating:
    • shouldComponentUpdate()
    • render()
    • componentDidUpdate()
  • Unmounting:
    • componentWillUnmount()

Lifecycle Functional Component (useEffect Hook):

import { useEffect } from 'react';

function Example() {
  useEffect(() => {
    // Kode untuk mounting/updating
    console.log('Component mounted/updated');
    
    return () => {
      // Cleanup untuk unmounting
      console.log('Component will unmount');
    };
  }, []); // Dependency array
}

Hooks

Hooks memungkinkan penggunaan state dan fitur React lainnya dalam functional components.

Hooks Dasar:

  • useState - Manajemen state 
    const [state, setState] = useState(initialValue);
  • useEffect - Side effects 
    useEffect(() => { /* effect */ }, [deps]);
  • useContext - Akses context
  • useReducer - State management kompleks

Hooks Tambahan:

  • useCallback - Memoize functions
  • useMemo - Memoize values
  • useRef - Referensi DOM/mutable values
  • useLayoutEffect - Efek sebelum paint

Custom Hooks:

function useCounter(initialValue) {
  const [count, setCount] = useState(initialValue);
  
  const increment = () => setCount(c => c + 1);
  const decrement = () => setCount(c => c - 1);
  
  return { count, increment, decrement };
}

// Penggunaan
function App() {
  const { count, increment } = useCounter(0);
  return <button onClick={increment}>{count}</button>;
}

Event Handling

Event handling di React mirip dengan DOM, tetapi dengan beberapa perbedaan sintaks.

Perbedaan Utama:

  • Nama event camelCase (onClick bukan onclick)
  • Handler berupa function bukan string
  • Harus call preventDefault() secara eksplisit

Contoh Event Handling:

function Button() {
  function handleClick(e) {
    e.preventDefault();
    console.log('Button clicked');
  }

  return (
    <button onClick={handleClick}>
      Click me
    </button>
  );
}

Event Handling dengan Parameter:

function List() {
  const items = ['Item 1', 'Item 2', 'Item 3'];
  
  return (
    <ul>
      {items.map((item, index) => (
        <li key={index} onClick={() => handleClick(item)}>
          {item}
        </li>
      ))}
    </ul>
  );
}

Form Handling

Form di React dikontrol melalui state (controlled components).

Controlled Components:

function Form() {
  const [name, setName] = useState('');
  
  const handleSubmit = (e) => {
    e.preventDefault();
    alert('Name: ' + name);
  };

  return (
    <form onSubmit={handleSubmit}>
      <label>
        Name:
        <input 
          type="text" 
          value={name}
          onChange={(e) => setName(e.target.value)}
        />
      </label>
      <button type="submit">Submit</button>
    </form>
  );
}

Form Kompleks:

function Form() {
  const [formData, setFormData] = useState({
    username: '',
    password: '',
    remember: false
  });

  const handleChange = (e) => {
    const { name, value, type, checked } = e.target;
    setFormData(prev => ({
      ...prev,
      [name]: type === 'checkbox' ? checked : value
    }));
  };

  // render form...
}

Routing

React Router adalah library standar untuk routing di aplikasi React.

Setup Dasar:

npm install react-router-dom

Contoh Implementasi:

import { BrowserRouter, Routes, Route, Link } from 'react-router-dom';

function App() {
  return (
    <BrowserRouter>
      <nav>
        <Link to="/">Home</Link>
        <Link to="/about">About</Link>
      </nav>
      
      <Routes>
        <Route path="/" element={<Home />} />
        <Route path="/about" element={<About />} />
      </Routes>
    </BrowserRouter>
  );
}

Dynamic Routing:

<Route path="/users/:id" element={<UserProfile />} />

// Di komponen UserProfile
import { useParams } from 'react-router-dom';

function UserProfile() {
  const { id } = useParams();
  return <div>User ID: {id}</div>;
}

Context API

Context menyediakan cara untuk membagikan data melalui komponen tree tanpa harus pass props secara manual.

Membuat Context:

import { createContext, useContext } from 'react';

const ThemeContext = createContext('light');

function App() {
  return (
    <ThemeContext.Provider value="dark">
      <Toolbar />
    </ThemeContext.Provider>
  );
}

function Toolbar() {
  const theme = useContext(ThemeContext);
  return <div>Current theme: {theme}</div>;
}

Context dengan State:

function App() {
  const [theme, setTheme] = useState('light');
  
  return (
    <ThemeContext.Provider value={{ theme, setTheme }}>
      <Toolbar />
    </ThemeContext.Provider>
  );
}

Redux

Redux adalah state management library untuk aplikasi JavaScript yang kompleks.

Konsep Dasar Redux:

  • Store: Tempat penyimpanan state global
  • Actions: Objek yang mendeskripsikan perubahan
  • Reducers: Fungsi yang menentukan perubahan state

Setup Redux dengan React:

npm install @reduxjs/toolkit react-redux

Contoh Implementasi:

// store.js
import { configureStore } from '@reduxjs/toolkit';

export default configureStore({
  reducer: {
    counter: counterReducer
  }
});

// App.js
import { Provider } from 'react-redux';
import store from './store';

function App() {
  return (
    <Provider store={store}>
      <Counter />
    </Provider>
  );
}

Testing

Testing adalah bagian penting dalam pengembangan aplikasi React.

Tools Testing:

  • Jest: Testing framework (sudah termasuk di CRA)
  • React Testing Library: Untuk testing komponen React
  • Cypress: End-to-end testing

Contoh Testing Komponen:

import { render, screen } from '@testing-library/react';
import Button from './Button';

test('renders button with text', () => {
  render(<Button>Click me</Button>);
  const buttonElement = screen.getByText(/click me/i);
  expect(buttonElement).toBeInTheDocument();
});

Testing Custom Hook:

import { renderHook, act } from '@testing-library/react-hooks';
import useCounter from './useCounter';

test('should increment counter', () => {
  const { result } = renderHook(() => useCounter());
  
  act(() => {
    result.current.increment();
  });
  
  expect(result.current.count).toBe(1);
});
